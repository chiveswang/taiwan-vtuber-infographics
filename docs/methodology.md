# Methodology

本文件說明公開資料與圖表的產生方式。正式公開前，任何資料都應先通過 privacy review。

## 資料流程

1. 從既有專案或公開來源取得候選資料。
2. 移除禁止欄位與高風險個人資訊。
3. 將可能敏感的個人層級資料轉成 aggregate data。
4. 保留 source URL、last verified 與資料處理說明。
5. 產出 derived data 與 charts。
6. 檢查圖表是否可能反推出個人作息、身份或敏感狀態。

## 發布標準

資料或圖表需符合：

- 不包含原始 tracking data。
- 不包含禁止欄位。
- 不支援即時追蹤。
- 不揭露個人層級敏感時間序列。
- 可被修正、移除或重新產出。

## 圖表建議

- 優先使用 aggregate trends。
- 優先使用月份、季度或年度級距。
- 避免個人排名與精細時間線。
- 樣本太小時不發布或合併分類。
